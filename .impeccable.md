# Impeccable — project design context

## Design Context

### Users
Developers running and supervising multiple AI coding agents at once from a Tauri
desktop app ("Agent Mission Control"). Used at the desk during focused work; the
operator is glancing between live agent terminals, triaging which agent needs input,
and organizing work by project. Power users, keyboard-driven, long sessions.

### Brand Personality
Precise, instrumented, calm-under-load — a control room, not a toy. Three words:
**instrumented, composed, fast.** The UI should feel like trustworthy equipment:
quiet by default, legible at a glance, with sharp accents only where attention is
warranted.

### Aesthetic Direction
Dark "deep-space" theme (near-black surfaces `--space-*` with a faint cool tint),
established in `src/lib/styles/tokens.css`. Type: **Space Grotesk** (display),
**Hanken Grotesk** (sans/body), **JetBrains Mono** (labels, numerics, paths).
Mono uppercase micro-labels for field/section headers. Accent is a restrained blue
(`--blue-*`); status colors (orange = needs you, red = destructive) are rare and
meaningful. Hairline borders, subtle elevation, short motion (`--dur-*`).
Anti-reference: generic SaaS dashboards, glassmorphism, neon-on-dark.

### Design Principles
1. **Honor the system.** Use the existing tokens (color/space/type/motion); never
   hard-code values that a token already expresses.
2. **Hierarchy through restraint.** One primary action per surface; everything else
   ghost/secondary. Accents earn their rarity.
3. **Legible at a glance.** Mono for labels/paths/numerics; clear type contrast;
   light-on-dark gets extra line-height.
4. **Quiet until it matters.** Motion and color signal state changes, not decoration.
5. **Perceptually-uniform color.** OKLCH for any derived/auto color, with a
   readability floor on dark surfaces.
